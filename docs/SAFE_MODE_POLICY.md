# Safe Mode Policy

## Principle
Prefer no harmful change over uncertain high-impact change.

## Safe Mode Triggers
- Missing measurement integrity
- Provisional evidence on high-blast-radius action
- Unclear rollback path
- Repeated QA failures in last 2 cycles

## Safe Mode Actions
- Freeze production-impacting updates.
- Route to human-led review.
- Run low-risk diagnostics and simulation only.
- Reopen execution only after gate pass and explicit approval.
