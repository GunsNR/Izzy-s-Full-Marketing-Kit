# Quality Gates

## Gate 1 — Evidence Gate
All meaningful claims, recommendations, and change proposals must include label: **Verified / Probable / Provisional**.

## Gate 2 — Risk Gate (Human Review Required)
Required before execution for:
- site-wide SEO changes
- major tracking changes
- major schema/internal-link changes
- pricing/offer changes
- significant paid budget changes
- high-impact bid strategy changes
- compliance-sensitive claims
- CRM automations with data corruption risk
- high-blast-radius retention/upsell changes
- final brand-positioning shifts

## Gate 3 — Delivery Gate
All major outputs must include:
1. objective
2. assumptions
3. source clarity
4. implementation steps
5. dependencies
6. rollback note (when relevant)
7. measurement plan
8. next action

## Enforcement
- Primary enforcement: `skills/quality-control-auditor/SKILL.md`.
- Orchestrator must block routing of outputs that fail any gate.
