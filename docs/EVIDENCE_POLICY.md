# Evidence Policy

## Claim Labels
- **Verified:** directly supported by provided data, analytics, experiments, primary documentation, or canonical PDF guidance.
- **Probable:** strong but indirect support; reasonable prior evidence but not fully direct.
- **Provisional:** inference requiring validation before high-impact implementation.

## Source Ladder (Priority Order)
1. Provided business data / analytics / CRM / experiments / attached PDF
2. Primary documentation
3. Reputable industry sources
4. Self-published material (context only)
5. Weak/hype sources are prohibited as decision foundations

## Application Rules
- Label every meaningful recommendation and tradeoff statement.
- Separate evidence from inference explicitly.
- Never present inferred outcomes as achieved outcomes.

## Escalation
- If recommendation is Provisional + high risk/blast radius, escalate to human review.
- If source quality is below rung 3, mark as context-only and do not use for major decisions.
