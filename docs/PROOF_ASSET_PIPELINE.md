# Proof Asset Pipeline

## Purpose
Standardize evidence artifacts that improve delivery quality, internal learning, sales enablement, retention, and expansion.

## Pipeline Stages
1. **Capture:** raw experiment/channel/change artifacts.
2. **Validate:** evidence label + source ladder compliance.
3. **Package:** operator-ready summary using standard template.
4. **Distribute:** attach to reporting, sales enablement, and retention workflows.
5. **Archive:** tag by offer, ICP, channel, date, confidence.

## Required Asset Templates
- Experiment write-up
- SEO change log
- Landing page test result
- Reporting narrative
- Case-study source capture
- Sales proof summary

## Standard Template Fields
- Objective
- Context
- Change made
- Evidence label (Verified/Probable/Provisional)
- Data window
- Results
- Risks/limitations
- Next action
- What we learned / What we will do next / What we will stop
